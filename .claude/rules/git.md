# Git Workflow

## Branch Naming

Use prefixes: `feat/`, `fix/`, `ci/`, `docs/`, `refactor/`, `perf/`

Example: `feat/add-oauth-support`, `fix/token-refresh-bug`

## Commit Messages

Use conventional commits format:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation only
- `ci:` - CI/CD changes
- `refactor:` - Code refactoring
- `perf:` - Performance improvement
- `chore:` - Maintenance tasks

Example: `feat: add Google OAuth integration`
